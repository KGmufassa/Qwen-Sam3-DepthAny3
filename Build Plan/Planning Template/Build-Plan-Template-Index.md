# Build Plan Template Index

## Purpose

This folder contains a reusable planning template derived from the existing `Build Plan` stack.

It is intended to turn a raw app idea into:

- a scoped PRD
- frozen strategic and technical assumptions
- vertical implementation slices
- slice task lists
- implementation tickets

The template keeps the planning depth of the current repository while removing product-specific assumptions.

---

## Recommended Planning Flow

Use this template in the following order:

1. complete the app idea intake
2. write the PRD and classify the build
3. freeze Stage 1 through Stage 4 plans
4. define the implementation roadmap and vertical slices
5. create one implementation plan per slice
6. derive one task list per slice
7. derive one ticket set per slice

## Output Structure

For each new app idea, create a planning stack with these layers:

- `PRDs`
- `1-4 Stage Planning/Stage-1`
- `1-4 Stage Planning/Stage-2`
- `1-4 Stage Planning/Stage-3`
- `1-4 Stage Planning/Stage-4`
- `Planing Resources`
- `Slice Implementaion`
- `Task List`
- `Implementation Tickets`

---

## Execution Rule

Do not create slice tasks or tickets until the foundational assumptions are frozen.

If core scope, contracts, runtime architecture, or release boundaries change, reopen the affected planning stage before continuing implementation planning.
